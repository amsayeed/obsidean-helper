Python Data Processing

```python
import pandas as pd
import numpy as np
from datetime import datetime

# Load data
df = pd.read_csv('data.csv')

# Data cleaning
df.dropna(inplace=True)
df['date'] = pd.to_datetime(df['date'])

# Feature engineering
df['year'] = df['date'].dt.year
df['month'] = df['date'].dt.month

# Aggregation
summary = df.groupby(['year', 'month']).agg({
    'sales': 'sum',
    'quantity': 'mean',
    'customer_id': 'nunique'
}).reset_index()

print(summary.head())
```

SQL Query Template

```sql
-- Data Quality Check Query
WITH data_quality AS (
    SELECT 
        COUNT(*) as total_records,
        COUNT(DISTINCT customer_id) as unique_customers,
        SUM(CASE WHEN order_date IS NULL THEN 1 ELSE 0 END) as null_dates,
        SUM(CASE WHEN amount < 0 THEN 1 ELSE 0 END) as negative_amounts,
        MIN(order_date) as earliest_date,
        MAX(order_date) as latest_date
    FROM orders
    WHERE created_at >= CURRENT_DATE - INTERVAL '30 days'
)
SELECT * FROM data_quality;
```

Shell Script

```bash
#!/bin/bash
# Data Backup Script

# Configuration
BACKUP_DIR="/backups/$(date +%Y%m%d)"
DB_NAME="production"
S3_BUCKET="s3://company-backups"

# Create backup directory
mkdir -p "$BACKUP_DIR"

# Backup database
echo "Starting database backup..."
pg_dump $DB_NAME | gzip > "$BACKUP_DIR/db_backup.sql.gz"

# Sync to S3
echo "Uploading to S3..."
aws s3 sync "$BACKUP_DIR" "$S3_BUCKET/$(date +%Y%m%d)/"

# Cleanup old backups (keep last 7 days)
find /backups -type d -mtime +7 -exec rm -rf {} +

echo "Backup completed successfully!"
```

# 📊 Customer Entity

## Overview
Central entity representing customers in the system.

## Schema

| Field | Type | Constraints | Description |
|-------|------|-------------|-------------|
| customer_id | UUID | PK | Unique customer identifier |
| email | VARCHAR(255) | UNIQUE, NOT NULL | Customer email |
| created_at | TIMESTAMP | NOT NULL | Account creation time |
| status | ENUM | NOT NULL | Active/Inactive/Suspended |
| segment | VARCHAR(50) | | Customer segment |

## Relationships
- **Orders**: One-to-Many (customer_id → orders.customer_id)
- **Addresses**: One-to-Many (customer_id → addresses.customer_id)
- **Preferences**: One-to-One (customer_id → preferences.customer_id)

## Business Rules
1. Email must be verified before activation
2. Suspended customers cannot place orders
3. Segment updated based on purchase history

Architecture Overview

# 🏗️ Data Platform Architecture

## Overview
Modern cloud-native data platform built on AWS/GCP/Azure.

## Components

### 1. Data Ingestion Layer
- **Batch Processing**: Apache Airflow
- **Stream Processing**: Apache Kafka / Kinesis
- **API Integration**: Custom Python services
- **File Transfer**: AWS Transfer Family

### 2. Storage Layer
- **Raw Data Lake**: S3 / GCS / Azure Blob
- **Processed Data**: Delta Lake / Iceberg
- **Data Warehouse**: Snowflake / BigQuery / Redshift
- **Operational Store**: PostgreSQL / MongoDB

### 3. Processing Layer
- **Batch ETL**: Apache Spark / AWS Glue
- **Stream Processing**: Spark Streaming / Flink
- **Data Transformation**: DBT
- **Data Quality**: Great Expectations

### 4. Serving Layer
- **BI/Analytics**: Tableau / Power BI / Looker
- **Data API**: GraphQL / REST APIs
- **ML Platform**: SageMaker / Vertex AI
- **Data Catalog**: DataHub / Alation

## Security & Governance
- **Authentication**: SSO with SAML/OAuth
- **Encryption**: At-rest and in-transit
- **Access Control**: IAM roles and policies
- **Compliance**: GDPR, CCPA compliant
